# Best practices
- Always grant _Write_ access to your team;
- Add topics to [README.md](README.md) to provide a better experience to other developers and users. Frequent topics are `Instructions`, `Install`, and `Build`;
- Add or modify the [.gitignore](.gitignore) to exclude unwanted files and directories;
- Add or modify the [LICENSE](LICENSE) according to the project needs and the Digiwest policy;
- Prefer semantic [versioning](https://semver.org/) and [conventional commits](https://www.conventionalcommits.org/);
- _Issues_ must be allowed to ensure that other devs (and yourself) can add, track, and resolve issues related to this repository;
- For small and private projects or repositories, _Project_ and _Discussion_ are usually disabled;
- Ensure that branch protection is enabled to prevent issues;
